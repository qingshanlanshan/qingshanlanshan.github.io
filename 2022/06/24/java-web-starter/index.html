JavaWeb大作业遇到的一些问题

配置

环境：Nginx+Tomcat+JSP+Servlet+MySQL
VPS：vultr 上海连接日本服务器的RTT大约130~140
域名：namecheap
工具：IDEA，VS Code（Remote-SSH extension），Navicat，PuTTY

连接服务器上的MySQL

在MySQL安装完成后，新建用户并授权
在/etc/mysql/mysql.conf.d/mysqld.cnf中，修改bind-address = 127.0.0.1（或者直接注释掉）
开放ufw的3306端口

Nginx

将jsp等动态资源通过proxy_pass http://127.0.0.1:8080反代到Tomcat服务器，而静态资源不经过Tomcat，error page也直接交给nginx，理论上减轻了服务器的压力，实际上，笑死，我就一个vps，而且vcpu使用率一直个位数……nginx设置完之后记得nginx -s reload

jsp和servlet比较严重的问题是我不会js和json…用别的凑合下好像还行？

Tomcat

开一下8080端口，然后把项目带上lib打包放到tomcat/webapps/ROOT下，nginx也要改一下root，Tomcat的热部署应该是默认开启的
